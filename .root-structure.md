# Root Directory Structure Guide

**Last Updated**: June 25, 2026  
**Purpose**: Explain root-level files and directories for developers

## 📁 Directory Structure

### Applications & Packages
- **`apps/`** - 4 workspace applications
  - `api/` - Spring Boot 3 backend (SaaS)
  - `web/` - Next.js 14 admin panel
  - `desktop/` - Tauri 2 offline-first terminal
  - `print-agent/` - Node.js print service
- **`packages/`** - Shared libraries
  - `config/` - Shared eslint, prettier, tsconfig, vitest
  - `types/` - Shared TypeScript types
  - `print-core/` - Print logic library

### Infrastructure & Quality
- **`infra/`** - Deployment configs
  - `docker-compose.yml` - Local development environment
  - Terraform configs for cloud
- **`qa/`** - Test suites
  - `e2e/` - End-to-end tests
  - `contract/` - API contract tests
  - `load/` - Load testing
  - `chaos/` - Chaos engineering tests
- **`scripts/`** - Monorepo automation
  - `dev.sh` - Start all services
  - `validate.sh` - Full build validation
  - `db.sh` - Database management
- **`security/`** - Security scanning
  - OWASP ZAP configs
  - Security scanning scripts
- **`docs/`** - Project documentation
  - `architecture/` - ADRs (Architecture Decision Records)
  - `contributing/` - CODE_OF_CONDUCT, CONTRIBUTING
  - `development/` - Setup guides
  - `guides/` - Feature guides
  - `troubleshooting/` - Debugging guides
  - `security/` - Security policies

### Configuration & Meta
- **`.config/`** - Tool configuration files
  - `.editorconfig` - Editor settings
  - `.npmrc` - NPM registry config
  - `.prettierrc` - Code formatter settings
  - `.prettierignore` - Prettier ignore patterns
  - `.mcp.json` - MCP server configs
  - `sonar-project.properties` - SonarQube analysis config

- **`.github/`** - GitHub-specific (workflows, templates)
- **`.vscode/`** - VS Code settings
- **`.claude/`** - Claude Code settings
  - `plans/` - Implementation plans
  - `memory/` - Auto-memory files
  - `settings.json` - Claude Code configuration

- **`.env.example`** - Example environment variables (commit this)
- **`.env`** - Local environment secrets (gitignored)
- **`.gitignore`** - Git ignore patterns
- **`pnpm-workspace.yaml`** - Monorepo workspace definition
- **`package.json`** - Root package scripts
- **`pnpm-lock.yaml`** - Dependency lock file
- **`README.md`** - Project overview (entry point)
- **`CLAUDE.md`** - Development rules & standards
- **`LICENSE`** - Apache 2.0 license

## 🚀 Quick Commands

```bash
# Development
pnpm dev           # Start all services
pnpm validate      # Full build + test validation
pnpm test          # Run all tests

# Database
pnpm db:up         # Start PostgreSQL
pnpm db:down       # Stop PostgreSQL

# Build
pnpm build:api     # Build backend
pnpm build:web     # Build web
pnpm build:desktop # Build desktop
```

## 📝 Adding New Files/Dirs

### New app?
- Create in `apps/<app-name>/`
- Add to `pnpm-workspace.yaml`
- Update this guide

### New configuration file?
- Place in `.config/` if tool-specific
- Update `.gitignore` if needed
- Document in CLAUDE.md if affects development

### New documentation?
- Place in `docs/` with appropriate subdirectory
- Link from main README if important
- Update this guide

### New automation script?
- Place in `scripts/`
- Make executable: `chmod +x scripts/my-script.sh`
- Document in README or this guide

## 🎯 Structure Philosophy

**Goal**: Keep root directory clean and organized while maintaining clarity about project structure.

**Principles**:
- Related files grouped in directories
- Configuration separated to `.config/`
- Documentation in `docs/`
- Code in `apps/` and `packages/`
- One entry point: README.md
